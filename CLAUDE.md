## 학습 진행 방식 (상세)

사용자가 공부를 시작하면 **이 순서대로** 진행할 것:

### 1단계: 학습 목표 제시
- 이 주제에서 배울 핵심 내용 소개
- 끝나면 뭘 할 수 있게 되는지 설명
- **예지보전 시스템 배포에서 어떻게 쓰이는지** 언급

### 2단계: 동기부여
- 왜 이게 필요한지, 실제로 어디에 쓰이는지 설명
- **"내 PC에서는 되는데 서버에서는 안 돼요" 문제 해결** 관점

### 3단계: 핵심 개념 설명
- 이론을 쉽게 풀어서 설명
- 필요하면 그림이나 다이어그램 (ASCII art) 활용
- 명령어는 하나씩 차근차근

### 4단계: 실습 예제
- 직접 따라할 수 있는 예제 제공
- **실제 Docker 명령어를 실행**하면서 학습
- 결과가 어떻게 나오는지 미리 알려주기

### 5단계: 스스로 해보기
- 연습 문제 2-3개 제시
- 힌트는 요청하면 제공
- 정답은 사용자가 시도한 후에 확인

### 6단계: 퀴즈
- 이해도 확인을 위한 간단한 퀴즈 2-3개
- 개념 확인 + 실전 문제 섞어서

### 7단계: 정리
- 오늘 배운 핵심 개념/명령어 요약
- 다음 주제 예고

## 복습 시스템 (3단계 진행)

```
review/
├── 1-new/terms.md       ← 새로 배운 용어/개념
├── 2-practicing/terms.md ← 복습 중 (1차 퀴즈 통과)
└── 3-mastered/terms.md   ← 완벽 암기 (2차 퀴즈 통과)
```

### 복습 프로세스

1. **매 수업 시작 시**: `1-new/terms.md`와 `2-practicing/terms.md`에서 랜덤 퀴즈 출제
2. **퀴즈 결과에 따라 승급**:
   - 2-practicing에서 정답 → `3-mastered`로 이동
   - 오답 → 현재 단계에 유지
3. **챕터 학습 후**: 틀린 것/몰랐던 것을 `2-practicing`에 추가
4. **새로 배운 개념**: `1-new`에 추가 → 다음 세션 퀴즈에서 통과하면 `2-practicing`으로

## 노트 작성 가이드 (튜토리얼 스타일)

notes/ 폴더의 노트는 **"이걸 보면 남에게 설명할 수 있다"** 수준으로 작성할 것.

### 포함할 내용
- **핵심 개념**: 이론 정리 (왜 필요한지, 뭔지)
- **다이어그램**: ASCII art나 표로 흐름/구조 시각화
- **명령어/설정 정리표**: 한눈에 보는 요약
- **실습 가이드**: 따라하기 좋은 순서 (복붙 가능하게)
- **주의사항 & 흔한 실수**: 학습 중 겪은 실수, 헷갈리는 포인트

### 작성 원칙
- 기존 notes/ 구조에 맞춰 저장
- 개념 설명 → 실습 → 주의사항 순서로 구성
- 실습 예제는 **처음부터 끝까지 따라할 수 있게** 작성

## 학습 완료 후 할 일

- `logs/` 폴더에 학습 일지 기록
- `notes/` 폴더에 학습 노트 작성
- `review/` 폴더에 새로 배운 용어/틀린 것 추가
- README.md의 체크리스트 업데이트 ([ ] → [x])
- README.md의 학습 일지 테이블에 기록 추가
- 커밋 메시지 형식: `docs: [주제] 학습 완료 - [주요 키워드]`
- **상위 study/ 폴더는 별도 git repo가 아니므로 커밋할 필요 없음.**

## 실행 검증

Docker 실습은 반드시 **실제로 명령어를 실행**해서 결과를 확인할 것:
- `docker ps`, `docker images` 등으로 상태 확인
- 에러가 나면 원인 분석 후 해결
- 실습 후 생성한 컨테이너/이미지는 정리 (`docker system prune`)

## 학습 마인드셋

- 학습 목표는 **"최소한 이건 알아야 해"의 기준**이지, 전부가 아님

- 에러가 나는 게 정상 — 에러 메시지 읽는 습관 들이기
- 나중에 다시 돌아와서 복습해도 됨
